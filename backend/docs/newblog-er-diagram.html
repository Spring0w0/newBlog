NewBlog 数据库 ER 图

基于当前 Flyway 迁移脚本生成。当前数据库共有 17 张表：16 张业务表，另有 1 张 Flyway 自动维护的迁移历史表。

实线表示数据库真实外键；虚线表示业务逻辑关联，目前没有外键约束。当前唯一真实外键是 blog_images.blog_id → blog_posts.id。

ER 图

表用途速览

博客 blog_posts

文章主表，保存 slug、标题、Markdown 正文、摘要、标签、分类、封面、隐藏状态和浏览数。

  • 唯一入口：slug
  • 正文：markdown
  • 标签：tags JSON

博客 blog_images

文章图片表，一篇文章可以有多张图片。当前唯一真实外键指向 blog_posts.id。

  • blog_id 外键
  • sha256 用于去重
  • storage_path 记录本地存储路径

博客 categories

分类字典表，保存分类名称和排序。当前与文章通过分类名称做逻辑关联。

  • name 唯一
  • sort_order 控制展示顺序

博客 likes

点赞流水表，记录文章 slug、IP、User-Agent 和时间，用来统计点赞和做 1 小时防刷。

  • 逻辑关联：blog_slug → blog_posts.slug
  • 索引支持 IP 时间窗口查询

站点 site_config

站点单例配置表，保存网站标题、描述、头像、favicon、备案号、帽子文案和主题色。

  • id = 1 单例
  • theme JSON 保存颜色变量

站点 card_styles

首页卡片布局表，保存各卡片宽高、偏移、排序和是否启用。

  • card_key 唯一
  • 对应前端首页拖拽布局

站点 art_images

首页装饰图，用于 ArtCard 或首页视觉内容。

  • url 图片地址
  • sort_order 控制顺序

站点 background_images

首页背景图资源表。

  • id 字符串主键
  • url 图片地址

站点 social_buttons

首页社交按钮配置，支持 GitHub、邮箱、B 站、知乎等不同类型。

  • type 平台类型
  • value 链接或值
  • label 展示名

内容 bloggers

博主收藏列表。

  • 头像：avatar_url
  • 评分：stars

内容 projects

项目展示列表。

  • 截图：image_url
  • 链接：url

内容 shares

分享链接或友情链接列表。

  • Logo：logo_url
  • 链接：url

内容 pictures

图片画廊，单条记录可以保存一组图片。

  • images JSON 保存多图 URL
  • description 描述

内容 snippets

一言、摘录或代码片段内容。

  • content 正文
  • sort_order 排序

内容 about

关于页面单例内容。

  • id = 1 单例
  • content 可存 Markdown

系统 users

管理员账户表，后续登录/JWT 会用它。

  • password_hash 存 BCrypt 哈希
  • role 目前默认 ADMIN

运维 flyway_schema_history

Flyway 自动创建和维护，记录哪些数据库迁移脚本已经执行。

  • 不要手动改这张表
  • 当前记录 V1、V2 已执行

关系说明

关系 类型 说明
blog_images.blog_id → blog_posts.id 真实外键 一篇文章可以有多张图片；删除文章时，图片记录会级联删除。
blog_posts.category ↔ categories.name 业务逻辑关联 文章分类当前存名称，不存 category_id。这样兼容前端现有 JSON 数据，但重命名分类时需要同步更新文章。
likes.blog_slug ↔ blog_posts.slug 业务逻辑关联 点赞按文章 slug 记录，不直接外键到文章 id。好处是迁移旧数据方便；坏处是改 slug 时需要同步点赞数据。
site_config 与首页相关表 业务聚合 站点配置接口通常会聚合 card_styles、art_images、background_images、social_buttons 返回给前端。
flyway_schema_history 运维表 Flyway 维护迁移版本，与业务表无业务关系。

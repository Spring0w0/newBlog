NewBlog 目标架构图

这张图描述从现有 Next.js 博客改造成 Java 后端后的整体结构：前端继续负责页面和交互， 后端集中承接认证、内容管理、图片上传、数据库读写、缓存和迁移，生产环境按单机轻量部署设计。

Next.js 前端 Nginx 反向代理 Spring Boot 3 MySQL 8 本地图片存储

一、部署与运行架构

目标是把原来直接读写 GitHub 的内容管理能力收敛到后端 API。浏览器只访问前端页面和公开资源， 前端通过 API 调用 Java 服务；Java 服务负责 MySQL、文件系统、缓存和初始化迁移。

入口与代理 Java 后端 持久化 本地文件 运维能力

二、后端分层架构

后端按照常规 Spring Boot 单体服务来落地：接口层只做参数和响应，业务层组织事务和规则， 数据层用 MyBatis-Plus 访问 MySQL，缓存、迁移、认证和上传作为横向能力接入。

三、模块职责

这些模块就是后面写实体、Mapper、Service 和 Controller 时的自然分组。

Auth / User

负责后台登录、JWT 签发、密码校验和接口权限控制。

  • 对应表：users
  • 依赖：Spring Security、JJWT
Blog

负责文章列表、详情、发布、编辑、删除、封面和文章内图片索引。

  • 对应表：blog_posts、blog_images
  • 主要接口：读写文章、批量删除、上传图片
Category / Like

负责分类字典和文章点赞数据，支撑前端博客页筛选与互动。

  • 对应表：categories、likes
  • 分类可做 Caffeine 缓存
Site Config

负责首页站点信息、卡片布局、背景图、艺术图、社交按钮等配置。

  • 对应表：site_config、card_styles
  • 高频读取，适合本地缓存
Content Pages

负责友链、项目、分享、相册、代码片段、关于页面等内容型列表。

  • 对应表：bloggers、projects、shares
  • 还有：pictures、snippets、about
Media

负责图片上传、文件命名、目录落盘、访问 URL 返回和后续清理策略。

  • 存储：本地磁盘目录
  • 数据库只保存 URL、路径、元信息
Migration

负责数据库结构版本管理，服务启动时自动检查并执行未应用的 SQL。

  • 工具：Flyway
  • 记录表：flyway_schema_history
Ops / Docs

负责健康检查、接口文档、日志、生产环境启动参数和排障入口。

  • Actuator：健康检查
  • SpringDoc：Swagger UI

四、典型请求流

以“发布一篇带图片的博客”为例，后端会同时处理鉴权、文件、数据库和缓存刷新。

1 后台登录

前端提交账号密码，后端校验 users，返回 JWT。

2 上传图片

前端上传封面或正文图片，后端保存到本地目录并返回 URL。

3 保存文章

提交标题、slug、markdown、分类、标签、图片引用等元数据。

4 事务入库

写入 blog_posts，同步维护 blog_images 关联。

5 刷新读取

清理相关缓存，前台列表和详情页下次请求获得最新数据。

五、开发与生产环境

配置文件已经按 dev/prod 分离：开发环境直接连本机 Docker MySQL，生产环境用环境变量注入敏感配置。

dev 开发环境

用于 IDEA 本地调试和接口开发。

配置文件 application-dev.yaml
数据库 localhost:3306/newblog，Docker MySQL 映射到本机 3306
账号 root / 1234
图片目录 D:/develop/newBlog/uploads/blog-images

prod 生产环境

用于云服务器单机部署，避免把密码写死在仓库里。

配置文件 application-prod.yaml
数据库 DB_URL、DB_USERNAME、DB_PASSWORD 环境变量
运行方式 java -jar，建议用 systemd 守护进程
图片目录 BLOG_UPLOAD_ROOT，默认可用 /data/blog-images
